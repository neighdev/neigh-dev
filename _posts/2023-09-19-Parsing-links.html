{% raw %}I've found that Jekyll transforms pages to have different paths in the browser, which would be annoying if it Jekyll couldn't resolve them for me. Which is done with {% link yourLinkHere %}

So I slapped {% link yourLinkHere %}  around all the relative links in the documents and I thought I was done with it.{% endraw %}

However, it turns out that all the links should then be relative from the root directory in order for Jekyll to resolve them. 

So I fixed that, however! It turns out that spaces in filenames are not allowed.

So I had to check all filenames for spaces and replace them with dashes, then filter all relative links and remove URLEncoding, and replace the spaces that are left over with dashes.

I should probably have removed url encoding when I started working with directory paths to begin with, but eh.

If it works it works.

If it doesn't work.. Well. It didn't.
